# 📚 使用示例

本文档提供AI对话导出工具的使用示例和最佳实践。

## 🎬 快速演示

### 示例1：基础对话导出

**输入对话：**
```
用户：请帮我写一个Python函数，计算两个数的和

AI：当然，这是一个简单的Python函数：

```python
def add_numbers(a, b):
    return a + b
```

用户：谢谢！这个函数能处理浮点数吗？

AI：可以的，这个函数可以处理整数和浮点数。
```

**处理步骤：**
1. 粘贴对话到输入框
2. 点击"自动清理"按钮
3. 系统识别到2条用户消息和2条AI消息
4. 选择要导出的消息（默认全选）
5. 设置标题为"Python编程对话"
6. 点击"生成PDF"并下载

**输出结果：**
- 生成包含封面的PDF文档
- 包含4条对话消息
- 代码块正确高亮显示
- 自动添加统计信息

  用户： 解释一下Python中的装饰器


AI：装饰器是Python的高级特性，用于修改函数或类的行为。

示例：

```python
def my_decorator(func):
    def wrapper():
        print("Before function call")
        func()
        print("After function call")
    return wrapper

@my_decorator
def say_hello():
    print("Hello!")

say_hello()
```


  用户：  装饰器有什么实际应用场景？


AI：  常见应用：
  - 日志记录
  - 性能测试
  - 权限验证
  - 缓存
```

**工具处理：**
- ✅ 自动清理多余的空格和空行
- ✅ 保留代码块的正确缩进
- ✅ 识别Markdown列表格式
- ✅ 正确分割消息边界

## 🎯 最佳实践

### 1. 对话格式建议

**推荐格式：**
```
用户：你的问题
AI：AI的回答（可能包含代码块）

用户：后续问题
AI：后续回答
```

**要避免的格式：**
- 缺少角色标记
- 代码块没有使用三个反引号
- 过多的空行和空格

### 2. 代码块处理

**正确：**
```python
def example():
    return "Hello"
```

**要避免：**
```
def example():
    return "Hello"  # 缺少代码块标记
```

### 3. 长对话处理

对于长对话（超过20条消息）：
1. 考虑分批次导出
2. 使用"按角色选择"功能快速筛选
3. 生成多个PDF文件分别保存

### 4. 文件命名建议

使用有意义的文件名：
- ✅ `Python_作业对话_20250408.pdf`
- ✅ `AI编程辅导_函数教学.pdf`
- ❌ `未命名.pdf`
- ❌ `新建文档.pdf`

## 🔧 高级功能使用

### 1. 文件上传功能

支持上传 `.txt`, `.md`, `.log` 格式文件：
1. 点击"或上传文本文件"
2. 选择文件
3. 系统自动加载内容
4. 继续后续处理步骤

### 2. 批量选择操作

提供多种选择方式：
- **全选**：选择所有消息
- **全不选**：取消所有选择
- **仅选用户**：只选择用户消息
- **仅选AI**：只选择AI消息

### 3. 自定义PDF设置

在侧边栏可以调整：
- 页面尺寸（A4/Letter）
- 是否包含封面
- 是否包含统计信息
- 页边距设置

### 4. 对话统计

工具自动生成统计信息：
- 总消息数量
- 用户消息数量
- AI消息数量
- 总字符数
- 平均消息长度

## 🚨 常见问题解决

### 问题1：消息识别不正确

**症状：** 用户和AI消息角色识别错误

**解决方案：**
1. 检查对话是否包含明确的角色标记
2. 确保使用支持的格式（用户：/AI：）
3. 可以手动编辑清理后的文本

### 问题2：代码块显示异常

**症状：** 代码没有正确高亮或格式混乱

**解决方案：**
1. 确保代码使用三个反引号包裹
2. 指定代码语言：```python
3. 避免在代码块中使用制表符，使用空格缩进

### 问题3：PDF生成失败

**症状：** 点击生成按钮后没有反应或报错

**解决方案：**
1. 检查是否选择了要导出的消息
2. 确保填写了文档标题
3. 检查系统内存是否充足
4. 尝试减少导出消息数量

### 问题4：界面显示异常

**症状：** 界面布局混乱或样式丢失

**解决方案：**
1. 清除浏览器缓存
2. 尝试使用其他浏览器
3. 检查网络连接
4. 重启Streamlit应用

## 📊 性能优化建议

### 处理大量消息时：
1. **分批处理**：超过100条消息时考虑分批
2. **选择性导出**：只导出关键对话部分
3. **关闭实时预览**：在处理时关闭预览功能
4. **增加内存**：确保系统有足够可用内存

### 提高处理速度：
1. 使用最新版本的Python和库
2. 关闭其他占用资源的应用程序
3. 使用SSD硬盘存储
4. 增加系统内存
